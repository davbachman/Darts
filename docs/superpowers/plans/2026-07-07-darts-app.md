# Darts App Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert the visible HandThrow app into a dedicated Darts app with a realistic board and fully scored Practice, 301, Cricket, and Around the Clock modes.

**Architecture:** Keep the existing Three.js dart throwing scene and gesture pipeline, but add a darts rules layer that receives completed board impacts. Replace the visible variant menu with a darts mode menu while keeping non-darts source code in place. Extract standard dartboard scoring into its own module so both rules and rendering use the same board order and hit geometry.

**Tech Stack:** TypeScript, Three.js, Vite, Vitest, Playwright-based web-game verification.

---

- Create `src/game/dartboard.ts`: standard board order, hit areas, impact scoring, board-label formatting, and helpers for test impacts.
- Create `src/game/dart-modes.ts`: Practice, 301, Cricket, and Around the Clock state transitions.
